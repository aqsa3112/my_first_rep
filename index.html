Block-level Elements

A block-level element always starts on a new line, and the browsers automatically add some space (a margin) before and after the element. A block-level element always takes up the full width available (stretches out to the left and right as far as it can). Two commonly used block elements are: < p > and <div>

  1. The < p >element defines a paragraph in an HTML document.
  2. The < div > element defines a division or a section in an HTML document.

Inline Elements

An inline element does not start on a new line. An inline element only takes up as much width as necessary.
e.g

  1. < a >
  2. < b >
  3. < br >
  4. < img >
  5. < big >
  6. < small >

What is favicon and how can we use it?

A favicon is a small icon associated with a website that appears in the browser's address bar, tabs, and bookmarks. It helps users identify and distinguish websites easily. To use a favicon, create a square image (usually 16x16 pixels or 32x32 pixels) and save it as a .ico file. Place the file in the root directory of your website, and browsers will automatically display it. You can also use a link tag in the HTML head to specify the favicon's location:
<link rel="icon" href="favicon.ico" type="image/x-icon" >

What is relative path and absolute path in html?

In HTML, a relative path and an absolute path are ways to specify the location of a resource, such as an image or a linked page.
Relative Path: A relative path specifies the location of a file or resource in relation to the current location. For example, if your HTML file is in a folder called "images," and you want to link to an image in the same folder, the relative path would be just the image filename (e.g., image.jpg). If the image is in a subfolder named "photos," the relative path would be photos/image.jpg.
Absolute Path: An absolute path provides the full URL or directory structure from the root of the file system or website. It includes the protocol (like http:// or https://), domain, and the full path to the resource. For example, https://example.com/images/image.jpg is an absolute path to an image on a website.

What is html attributes?

HTML attributes provide additional information about HTML elements.

  • All HTML elements can have attributes
  • Attributes provide additional information about elements
  • Attributes are always specified in the start tag
  • Attributes usually come in name/value pairs like: name="value"
The < a> tag defines a hyperlink. The href attribute specifies the URL of the page the link goes to:
<a href="https://www.w3schools.com">Visit W3Schools</a>

How title attribute works?

The title attribute in HTML is used to provide additional information about an element, typically when a user hovers over that element. It's commonly used with elements like < a> (anchor tags), < img> (image tags), and others.
<a href="https://www.example.com" title="Visit Example"> Click me</a>
The href attribute specifies the link's destination. The title attribute provides additional information that is displayed as a tooltip when the user hovers over the link.
So, when a user hovers over the "Click me" link, a small box containing the text "Visit Example" will appear, offering extra context or details related to the link. This is particularly useful for improving accessibility and user experience by giving users more information about the purpose or content of a link.
